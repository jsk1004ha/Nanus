# Nanus Production MVP Contract

This document defines what the current Nanus MVP guarantees after the production-hardening pass.

## Runtime guarantees

- WebSocket clients subscribe to persisted events only; they do not trigger execution.
- Runs are created as queued jobs and executed by the supervisor.
- Terminal statuses are explicit: `complete`, `degraded`, `failed`, `cancelled`.
- Fallback model/tool execution is never hidden as full success; it is reported as `degraded` through verification metadata.
- Assistant output is streamed through `assistant.message.delta` events and closed with `assistant.message.completed`.

## Document/RAG guarantees

- Text and uploaded documents are chunked and persisted under the backend data directory.
- `/api/chat` accepts `documentIds` and injects bounded retrieved document context into the run prompt while preserving the original user-visible message.
- Automatic document search is used when no explicit document ids are supplied and relevant chunks exist.

## Artifact guarantees

- Spreadsheet output is now a real OOXML `.xlsx` package generated by `spreadsheet_studio.py`, not an HTML `.xls` compatibility file.
- Artifact validation checks package-level integrity for PPTX and XLSX, and basic document structure for HTML/Markdown.
- `/api/runs/{id}/artifacts/validate` exposes artifact validation results for CI and UI trust badges.

## Browser and approval guarantees

- Browser snapshot is read-only and requires explicit approval when attached to a run.
- Private, loopback, link-local, multicast, and reserved IP targets are blocked by DNS resolution before fetching.
- Approval confirm/reject events are persisted and affect run state.

## Test coverage

The backend test suite covers:

- queued run execution and WebSocket event replay
- streamed answer delta events
- PPTX download package structure
- XLSX workbook package structure
- document upload and RAG context injection
- browser approval/reject flow and private target blocking
- payload size boundaries
- Codex bridge fallback and live bridge injection path
